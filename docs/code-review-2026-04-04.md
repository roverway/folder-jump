# FolderJump 代码质量复核报告

日期：2026-04-04

## 复核范围

本次复核覆盖以下方面：

- AutoHotkey v2 项目结构、模块边界与错误处理
- 不同文件管理器当前路径的识别逻辑
- 文件对话框中的路径跳转逻辑
- 配置、日志与文档的一致性

## 主要发现

### P1. Total Commander 面板方位语义不稳定

问题：

- `Total Commander` 的当前实现可以稳定拿到活动面板和非活动面板路径。
- 但“左 / 右”语义并没有来自稳定官方信号，之前依赖启发式推断，存在显示错误风险。

影响：

- 如果把启发式结果直接展示给用户，会误导用户选择。

建议：

- 保留 `active / inactive` 语义作为当前稳定输出。
- 若后续要恢复 `left / right`，应先确认是否存在可靠的官方窗口消息、控件标识或可验证信号。

当前状态：部分解决。

- UI 已不再显示不可靠的 `left / right`。
- `left / right` 的稳定建模仍未完成。

### P1. 文件对话框跳转原先过度依赖按键模拟

问题：

- 仅靠 `Ctrl+L`、`Alt+D`、粘贴、回车的方案，容易受到宿主程序、控件焦点和本地化差异影响。

影响：

- 某些“打开 / 另存为”对话框里，路径可能被写入错误控件，或者快捷键不生效。

建议：

1. 优先做控件级方案。
2. 次优方案使用快捷键。
3. 最后再退化为通用按键模拟。

当前状态：已改进但未彻底完成。

- 已实现“控件级优先、快捷键次之、通用按键兜底”的三层策略。
- 仍需继续增强现代 Common Item Dialog 下的控件识别和结果校验。

### P1. Directory Opus 路径识别不应长期依赖标题猜测

问题：

- 单靠窗口标题或控件文本猜路径，容易被自定义标题、多标签或布局变化干扰。

影响：

- 可能把非当前目录误识别为有效路径。

建议：

- 优先使用官方集成方式，如 `DOpusRT` 或脚本 / 自动化接口。
- 标题解析仅作为兜底。

当前状态：已改进。

- 当前实现已切换为 `DOpusRT /info ...,paths` 优先。
- 标题解析保留为回退方案。
- 仍需更多版本和布局验证。

### P2. 文件对话框识别不应主要依赖标题关键词

问题：

- 仅靠“打开 / 保存 / 另存为 / Open / Save”等标题关键词，容易在不同语言和宿主应用下误判。

建议：

- 优先基于控件结构识别文件对话框。
- 标题关键词作为辅助手段，而不是主判断依据。

当前状态：已解决。

- 当前识别已升级为控件结构优先，标题关键词兜底。

### P2. Windows 路径去重需要大小写归一化

问题：

- 在 Windows 下，`C:\Work`、`c:\work`、`C:\Work\` 本质上应视为同一路径。

建议：

- 去重时统一做大小写和尾部反斜杠归一化。

当前状态：已解决。

- 当前路径收集阶段已补上 Windows 语义下的归一化去重。

### P3. 文档与真实实现存在阶段性漂移

问题：

- 设计文档、配置模板和 README 在多次功能调整后，容易继续保留过期描述。

建议：

- 在功能收敛后及时同步 README、配置模板和技术设计文档。
- 对复核报告补充“已解决 / 未解决”附录，避免后续阅读时混淆状态。

当前状态：部分解决。

- README、配置模板、技术设计文档已同步一轮。
- 本报告现已补充状态说明。

## 关于不同文件管理器路径识别的建议

### Explorer

建议继续保持 COM 方案为主：

- `Shell.Application` 枚举路径稳定性较高。
- 窗口标题解析只保留为少数异常情况下的降级路径。

### Total Commander

建议保守处理语义：

- 当前可以可靠表达 `active / inactive`。
- 在没有稳定左右侧信号前，不建议重新对外显示 `left / right`。

### Directory Opus

建议继续官方接口优先：

- `DOpusRT /info ...,paths` 适合作为当前主方案。
- 后续如果需要更强能力，可再评估脚本 / 自动化接口是否值得引入。

## 关于文件对话框跳转的建议

当前最合理的策略是：

1. 控件级定位优先。
2. `Ctrl+L` / `Alt+D` 作为快捷键回退。
3. 最后再做通用按键模拟。

后续仍可继续增强的点：

- 针对现代 Common Item Dialog 做更精细的地址栏控件识别。
- 在提交路径后增加更可靠的结果校验，而不是仅依赖短暂延时。
- 对不同宿主程序维护少量兼容性特例，而不是无限扩展通用猜测逻辑。

## 已解决 / 未解决附录（2026-04-05）

### 已解决

- `Ctrl+G` 触发范围已收紧到文件对话框，不再在 `Total Commander`、Explorer、DOpus 主窗口中直接激活。
- 文件对话框跳转已改成三层策略：控件级优先、快捷键次之、通用按键兜底。
- `Directory Opus` 已改为 `DOpusRT /info ...,paths` 优先收集路径。
- 文件对话框识别已改为控件结构优先，标题关键词兜底。
- Windows 路径去重已加入大小写和尾部分隔符归一化。
- `XYplorer` 的“声明支持但未实现”状态已清理，不再继续误导。
- README、配置模板、技术设计文档已同步到当前实现一轮。

### 未完全解决

- `Total Commander` 的 `left / right` 语义仍未得到稳定、官方可验证的建模。
- 文件对话框控件识别和结果校验还有继续增强空间。
- `DOpusRT` 返回结构仍需在更多版本与布局下验证。
- 仓库中文本文件虽然已统一到 UTF-8 with BOM 方向，但仍建议后续做一次更系统的历史文件巡检。
